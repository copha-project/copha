# 内置模块注册指引
copha 接受第三方提交的模块注册申请，审核通过的模块将会被添加至 copha 内置模块列表。


### 注册方式
在 copha 位于 GitHub 的仓库上新开一个注册请求 issue（ [去提交](https://github.com/copha-project/copha/issues/new) ） 并提供以下信息：
- 模块名字
- 模块说明

注册模块的审核进度，反馈，结果会在该 issue 里进行。

### 注册模块的一些要求
1. 名字
	- 模块名字确保与已有的内置模块名不冲突
	- 尽可能有意义
	- 长度合适
2. 功能
	- 满足特定需求
3. 其他
	- 注意版权争端
	- 注意法规风险
